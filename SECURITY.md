# Security Policy

## Reporting a Vulnerability

Please do not open public GitHub issues for security problems.

Instead, report vulnerabilities privately to the maintainer with:

- a description of the issue
- affected files or flows
- reproduction steps
- impact assessment
- any suggested mitigation

## Scope

Security-relevant areas include:

- Bluetooth connection handling
- packet parsing
- data validation
- device discovery filters
- build and signing configuration

## Response

Best effort response will include:

1. acknowledgment
2. validation or clarification
3. remediation plan if confirmed
4. disclosure coordination where appropriate
